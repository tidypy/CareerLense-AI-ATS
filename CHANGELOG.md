### Fixed

#### 🔑 BYOK (Bring Your Own Key) — Routing & Fallback Overhaul

**Issue:** The API key field was non-functional in two critical ways:
1. Leaving the field **blank** still attempted to contact Google's API using the server-side key, rather than routing directly to the Local RTX GPU (hosted via TailScale).
2. When a user's **invalid or expired** API key was submitted, the failure was silently shown as a brief auto-dismissing SnackBar, giving no clear signal to the user that their key was the problem (not the app), and providing no automatic fallback.

**Fix — Backend (`backend/llm_service.py`):**
- Added typed `BYOKKeyError` exception for distinguishing user-key failures from system failures.
- `GoogleLLMClient` now catches specific Google API errors (`InvalidArgument`, `PermissionDenied`, `ResourceExhausted`, `Unauthenticated`) and re-raises as `BYOKKeyError` with human-readable reasons.
- Rewrote `generate_career_data` routing with three explicit paths:
  - **Blank key** → routes directly to `LocalLLMClient` (Local RTX via TailScale). Google is never contacted.
  - **`http://...` key** → treated as an LM Studio endpoint URL, routes to local.
  - **Google key provided** → attempts Google; `BYOKKeyError` propagates to the API layer.

**Fix — Backend (`backend/main.py`):**
- `_do_generate` catches `BYOKKeyError`, immediately falls back to `LocalLLMClient` within the same HTTP request (no second round-trip).
- On successful local fallback, returns `HTTP 200` with the generated HTML **plus** the response header `X-BYOK-Error: <reason>`.
- If both Google and Local fail: returns `HTTP 502` with both error details.

**Fix — Frontend (`frontend/lib/main.dart`):**
- After every `200` response, checks for the `x-byok-error` response header.
- If present: shows a **blocking `AlertDialog`** (`barrierDismissible: false`) with:
  - The exact failure reason from the backend
  - A note clarifying the result was generated by the Local RTX fallback
  - An "OK — I Understand" button the user must tap to proceed
- After dismissal: the generated result is displayed normally.
- Updated API key field label/hint to clarify that leaving it blank routes to Local RTX.

#### 🤖 LLM Service & Validation Guard ([#3](https://github.com/tidypy/CareerLense-AI-ATS/issues/3))

- **Fix — Backend (`backend/llm_service.py`):**

  - Implemented explicit type guards for `schema_model` to prevent runtime instantiation failures.
  - Added **Raw Output Logging**: The system now logs the exact LLM response (e.g., "Quota Exceeded") before a validation error occurs, enabling faster triage.
- **Added — Backend (`backend/verify_key.py`):**
  - New diagnostic script for testing Google Gemini and Local LLM (RTX/LM Studio) connectivity outside the main application loop.

#### 🎨 Dynamic Status Badge & Local Fallback UX ([#4](https://github.com/tidypy/CareerLense-AI-ATS/issues/4))

- **Fix — Frontend (`frontend/lib/main.dart`):**

  - **Intelligent Dialog**: Updated `_showByokErrorDialog` with "⌛ Quota Exhausted" detection and a premium informational box.
